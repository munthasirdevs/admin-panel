# Modal

A versatile modal dialog component with accessibility features, focus trap, and customizable content.

## Installation

```tsx
import { Modal } from '@components';
```

## Basic Usage

```tsx
import { useState } from 'react';
import { Modal, Button } from '@components';

function App() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <Button onClick={() => setIsOpen(true)}>Open Modal</Button>
      <Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
        <p>Modal content goes here...</p>
      </Modal>
    </>
  );
}
```

## Modal Sections

### With Title

```tsx
<Modal isOpen={isOpen} onClose={handleClose} title="Modal Title">
  <p>Modal content...</p>
</Modal>
```

### With Title and Subtitle

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}

  subtitle="Modal subtitle or description"

</Modal>
```

### With Custom Header

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}

      <Icon className="w-6 h-6" />
      <h2>Custom Header</h2>
    </div>
  }
>
  <p>Modal content...</p>
</Modal>
```

### With Footer

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}
  title="Confirm Action"
  footer={
    <>
      <Button variant="outline" onClick={handleClose}>Cancel</Button>
      <Button variant="primary" onClick={handleConfirm}>Confirm</Button>
    </>
  }

  <p>Are you sure you want to proceed?</p>
</Modal>
```

### Using Sub-components

```tsx

    <Modal.Subtitle>Custom subtitle</Modal.Subtitle>

  <Modal.Body>

  </Modal.Body>
  <Modal.Footer>
    <Button variant="outline">Cancel</Button>
    <Button variant="primary">Save</Button>
  </Modal.Footer>
</Modal>
```

## Behavior Options

### Close on Backdrop Click

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}
  closeOnBackdropClick={true} // default
>
  <p>Click outside to close</p>
</Modal>
```

### Close on Escape

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}
  closeOnEscape={true} // default
>
  <p>Press Escape to close</p>
</Modal>
```

### Hide Close Button

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}
  showCloseButton={false}
>
  <p>No close button visible</p>
</Modal>
```

### Prevent Body Scroll

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}
  preventBodyScroll={true} // default
>
  <p>Body scroll is prevented</p>
</Modal>
```

### Disable Focus Trap

```tsx
<Modal
  isOpen={isOpen}
  onClose={handleClose}
  trapFocus={false}
>
  <p>Focus is not trapped</p>
</Modal>
```

## Props

### Modal Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isOpen` | `boolean` | - | Whether the modal is open |
| `onClose` | `() => void` | - | Callback when modal should close |
| `title` | `ReactNode` | - | Modal title |
| `subtitle` | `ReactNode` | - | Modal subtitle |
| `header` | `ReactNode` | - | Custom header content |
| `footer` | `ReactNode` | - | Footer content |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl' \| 'full'` | `'md'` | Modal size |
| `showCloseButton` | `boolean` | `true` | Show/hide close button |
| `closeOnBackdropClick` | `boolean` | `true` | Close on backdrop click |
| `closeOnEscape` | `boolean` | `true` | Close on Escape key |
| `trapFocus` | `boolean` | `true` | Trap focus inside modal |
| `preventBodyScroll` | `boolean` | `true` | Prevent body scroll |
| `animationDuration` | `number` | `200` | Animation duration in ms |
| `ariaLabel` | `string` | - | Aria label for accessibility |
| `ariaLabelledBy` | `string` | - | Aria labelledby ID |
| `isLoading` | `boolean` | `false` | Loading state |
| `className` | `string` | - | Additional CSS classes |

## Accessibility

- Modal has `role="dialog"` and `aria-modal="true"`
- Focus is trapped inside the modal when open
- Escape key closes the modal by default
- Title is associated with `aria-labelledby`
- Close button has proper `aria-label`
- Body scroll is prevented when modal is open

## Best Practices

1. **Clear titles**: Always provide a descriptive title
2. **Appropriate sizing**: Use the smallest size that fits your content
3. **Action clarity**: Use footer for primary/secondary actions
4. **Loading states**: Show loading state for async operations
5. **Don't overuse**: Modals interrupt user flow - use sparingly

## Examples

### Confirmation Dialog

```tsx
function DeleteConfirmation({ isOpen, onClose, onConfirm }) {
  return (
    <Modal
      isOpen={isOpen}
      onClose={onClose}
      title="Delete Item"
      subtitle="This action cannot be undone"
      footer={
        <>
          <Button variant="outline" onClick={onClose}>Cancel</Button>
          <Button variant="danger" onClick={onConfirm}>Delete</Button>
        </>
      }
    >
      <p>Are you sure you want to delete this item?</p>
    </Modal>
  );
}
```

### Form Modal

```tsx
function EditUserModal({ isOpen, onClose, user }) {
  const [formData, setFormData] = useState(user);

  return (
    <Modal
      isOpen={isOpen}
      onClose={onClose}
      title="Edit User"
      footer={
        <>
          <Button variant="outline" onClick={onClose}>Cancel</Button>
          <Button variant="primary" onClick={handleSave}>Save Changes</Button>
        </>
      }
    >
      <form className="space-y-4">
        <Input
          label="Name"
          value={formData.name}
          onChange={(e) => setFormData({ ...formData, name: e.target.value })}
        />
        <Input
          label="Email"
          type="email"
          value={formData.email}
          onChange={(e) => setFormData({ ...formData, email: e.target.value })}
        />
      </form>
    </Modal>
  );
}
```
